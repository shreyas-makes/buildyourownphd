## 🚀 What is Build Your Own PhD?

**Build Your Own PhD** is a revolutionary AI-powered platform that transforms any educational content into a structured, accessible learning experience. Whether you're a researcher, student, or lifelong learner, our platform converts PDFs, videos, articles, and audio files into beautiful markdown documentation with AI-generated audio narration.

> 🎯 **Mission**: Democratize access to knowledge by making any content consumable, searchable, and accessible across all devices.

## 🚀 Quick Start

### 📋 Prerequisites

<div align="center">

[![Node.js](https://img.shields.io/badge/Node.js-18+-339933?style=flat-square&logo=node.js&logoColor=white)](https://nodejs.org/)
[![npm](https://img.shields.io/badge/npm-9+-CB3837?style=flat-square&logo=npm&logoColor=white)](https://npmjs.com/)
[![Git](https://img.shields.io/badge/Git-2.34+-F05032?style=flat-square&logo=git&logoColor=white)](https://git-scm.com/)

</div>

### ⚡ One-Click Setup

```bash
# Clone and setup in one command
curl -sSL https://raw.githubusercontent.com/buildyourownphd/buildyourownphd/main/scripts/setup.sh | bash
```

### 🛠️ Manual Setup

<details>
<summary><strong>Click to expand manual setup instructions</strong></summary>

#### 1️⃣ Clone the Repository

```bash
git clone https://github.com/buildyourownphd/buildyourownphd.git
cd buildyourownphd
```

#### 2️⃣ Install Dependencies

```bash
npm install
```

#### 3️⃣ Environment Configuration

```bash
# Copy environment template
cp env.example .env.local

# Edit with your API keys
nano .env.local
```

```env
# 🔐 Essential API Keys
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key

# 🤖 AI Services
OPENAI_API_KEY=your_openai_api_key
DEEPINFRA_API_KEY=your_deepinfra_api_key
ANTHROPIC_API_KEY=your_anthropic_api_key

# 🔊 Text-to-Speech (Optional)
ELEVENLABS_API_KEY=your_elevenlabs_api_key
```

#### 4️⃣ Database Setup

```bash
# Initialize database schema
npm run db:setup

# Run migrations
npm run db:migrate

# Seed with sample data (optional)
npm run db:seed
```

#### 5️⃣ Start Development

```bash
# 🚀 Start all services
npm run dev

# Or start individually:
npm run dev:web      # 🌐 Web app → http://localhost:3000
npm run dev:mobile   # 📱 Mobile app → Expo DevTools
npm run dev:backend  # ⚙️ API server → http://localhost:8000
```

</details>

### 🎉 You're Ready!

Open [http://localhost:3000](http://localhost:3000) and start building your personal PhD curriculum!

### 🏃‍♂️ **Getting Started with Development**

```bash
# 1. Fork the repo and clone your fork
git clone https://github.com/YOUR_USERNAME/buildyourownphd.git
cd buildyourownphd

# 2. Add upstream remote
git remote add upstream https://github.com/buildyourownphd/buildyourownphd.git

# 3. Install dependencies
npm install

# 4. Create a feature branch
git checkout -b feature/your-feature-name

# 5. Start development
npm run dev
```

### 🧪 **Testing Your Changes**

```bash
# Run all tests
npm run test

# Run specific test suites
npm run test:unit        # Unit tests
npm run test:integration # Integration tests
npm run test:e2e         # End-to-end tests
npm run test:mobile      # Mobile app tests

# Run tests in watch mode
npm run test:watch
```

### 📝 **Code Style & Standards**

We follow these coding standards:

- **TypeScript**: All new code must be TypeScript
- **ESLint**: Follow the configured ESLint rules
- **Prettier**: Auto-format code with Prettier
- **Conventional Commits**: Use conventional commit format
- **Testing**: Write tests for new features
- **Documentation**: Update docs for new features

### 🔄 **Pull Request Process**

1. **Update Documentation**: Add/update relevant documentation
2. **Add Tests**: Ensure your code is well-tested
3. **Follow Code Style**: Run `npm run lint` and `npm run format`
4. **Update Changelog**: Add your changes to CHANGELOG.md
5. **Describe Changes**: Write a clear PR description
6. **Link Issues**: Reference any related issues

### 🏷️ **Issue Labels**

- `bug` - Something isn't working
- `enhancement` - New feature or request
- `documentation` - Improvements to documentation
- `good first issue` - Good for newcomers
- `help wanted` - Extra attention is needed
- `question` - Further information is requested

</details>
